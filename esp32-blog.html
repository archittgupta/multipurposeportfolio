Taming the Beast: Controlling 220V with Logic Levels

Microcontrollers live in a safe, quiet world. An ESP32 operates at 3.3V logic. It’s precise, low-energy, and harmless. The real world, however, runs on 220V Mains AC (in India). It is noisy, dangerous, and powerful.

My goal was simple: Turn a "dumb" extension board into a smart one. I wanted to control my desk setup (monitors, lights, chargers) directly from my phone or a web dashboard, without buying expensive proprietary smart plugs.

⚠️ SAFETY WARNING: Mains voltage (110V/220V) can be lethal. Do not attempt to wire relays to mains power unless you understand isolation and electrical safety.

The Bridge: Relays & Isolation

You cannot connect an AC appliance to a microcontroller pin. The magic component that bridges these two worlds is the Relay.

For this project, I used a 4-Channel Relay Module. The concept is straightforward but ingenious:

  • The ESP32 sends a tiny 3.3V signal to the relay module.
  • An Optocoupler inside the module uses light to transmit that signal, keeping the high voltage electrically isolated from the low voltage logic (protecting the ESP32 from frying).
  • The electromagnet clicks on, closing the high-voltage circuit and powering the appliance.

The Brain: ESP32 Web Server

While I could have used Blynk or Home Assistant, I wanted to build the control logic myself. I programmed the ESP32 using the Arduino IDE to host an AsyncWebServer.

The ESP32 creates a local IP address. When I visit that IP on my phone, it serves a simple HTML interface with toggle buttons. Clicking a button sends an HTTP GET request to the ESP32, which flips the corresponding GPIO pin.

Key Learnings

The biggest challenge wasn't the code—it was the physical space. Fitting an ESP32, a bulky relay module, and a 5V buck converter (to step down 220V to power the ESP itself) inside a standard extension box is a puzzle of geometry and heat management.
